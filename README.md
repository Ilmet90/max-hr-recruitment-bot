# MAX HR Recruitment Bot

Шаблон чат-бота MAX и web-панели для кадрового подразделения организации.

Проект помогает публиковать вакансии, принимать отклики кандидатов, вести вопросы и сообщения, показывать условия работы или службы, управлять фотоальбомами и распределять отклики между сотрудниками кадрового подразделения.

## Возможности

- публичное меню для кандидатов в MAX;
- список вакансий и карточки вакансий;
- пошаговый отклик на вакансию;
- вопросы кандидатов;
- сообщения через бота;
- раздел условий работы или службы;
- фотоальбомы с отправкой изображений через MAX;
- локальная web-админка на FastAPI и Jinja2;
- сотрудники кадрового подразделения;
- роли и подтверждение доступа;
- принятие отклика в работу;
- назначение откликов сотрудникам;
- изменение служебного кода `/admin`;
- настройка организации, публичных текстов и цветов интерфейса;
- автозапуск через systemd.

## Требования

- Debian 12/13 или Ubuntu Server;
- Python 3;
- доступ в интернет для MAX API;
- токен MAX-бота.

## Быстрая установка

```bash
git clone <repo-url>
cd max-hr-recruitment-bot
sudo bash install.sh
```

Установщик задаст вопросы об организации, главной web-учётной записи, служебном коде `/admin`, MAX-токене, host и port web-админки.

## Первый вход

После установки откройте web-интерфейс:

```text
http://<IP_сервера>:<APP_PORT>
```

Логин задаётся при установке. Пароль задаётся при установке или генерируется автоматически, если оставить поле пустым.

## Настройка MAX-токена

Если токен был указан при установке, служба бота запускается автоматически.

Если токен не был указан:

1. Откройте файл `/opt/max-hr-recruitment-bot/.env`.
2. Заполните `MAX_BOT_TOKEN`.
3. Запустите службу:

```bash
sudo systemctl start max-hr-bot.service
sudo systemctl enable max-hr-bot.service
```

## Регистрация сотрудников

Сотрудник пишет боту:

```text
/admin <служебный код>
```

Заявка получает статус «Ожидает подтверждения». Главная учётная запись или начальник кадрового подразделения подтверждает пользователя в web-админке или через служебное меню `/staff`.

## Роли

- Главная учётная запись — резервный web-вход и полный доступ.
- Начальник кадрового подразделения — подтверждает сотрудников, управляет ролями, назначает отклики и делегирует полномочия.
- Сотрудник кадрового подразделения — принимает отклики в работу, видит свои и общие отклики, редактирует контент.
- Ожидает подтверждения — заявка на доступ ещё не одобрена.
- Отключён — доступ отключён.

## Настройка организации

Главная учётная запись может открыть «Настройки организации» и изменить:

- полное и краткое название организации;
- вышестоящую организацию;
- регион;
- название бота;
- тексты приветствия;
- контакты;
- сообщения кандидату;
- предупреждение о персональных данных;
- заголовки web-админки;
- цвета интерфейса.

После сохранения новые значения используются в web-админке, приветствии бота, сообщениях кандидату, уведомлениях, условиях и контактах.

Служебные имена systemd и папка проекта могут оставаться прежними.

## Управление службами

```bash
sudo systemctl status max-hr-admin.service
sudo systemctl status max-hr-bot.service
sudo systemctl restart max-hr-admin.service
sudo systemctl restart max-hr-bot.service
sudo journalctl -u max-hr-admin.service -f
sudo journalctl -u max-hr-bot.service -f
```

## Резервное копирование

В установленном проекте:

```bash
cd /opt/max-hr-recruitment-bot
sudo bash scripts/backup.sh
```

Архив включает SQLite-базу и загруженные файлы.

## Обновление

В установленном проекте:

```bash
cd /opt/max-hr-recruitment-bot
sudo bash scripts/update.sh
```

Скрипт останавливает службы, делает резервную копию базы, обновляет код через `git pull`, если репозиторий настроен, устанавливает зависимости, выполняет миграции и запускает службы.

## Безопасность

- Не публикуйте `.env`.
- Не храните токены и пароли в репозитории.
- Используйте сложные пароли.
- Меняйте служебный код `/admin`.
- Не открывайте web-админку в интернет без HTTPS и дополнительной защиты.
- Используйте VPN, SSH-туннель или внутреннюю сеть.
- Регулярно делайте резервные копии.
- При утечке MAX-токена перевыпустите токен.
- При увольнении сотрудника отключайте его учётную запись.
- Не размещайте в публичных текстах сведения ограниченного доступа, персональные данные сотрудников или служебную информацию, не предназначенную для публикации.
