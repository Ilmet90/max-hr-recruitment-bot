# MAX HR Recruitment Bot

Шаблон чат-бота MAX и web-панели для кадрового подразделения организации.

Проект помогает публиковать вакансии, принимать отклики кандидатов, вести вопросы и сообщения, показывать условия работы или службы, управлять фотоальбомами, распределять отклики между сотрудниками и обслуживать установленную систему через web-интерфейс.

## Возможности

- публичное меню кандидата в MAX;
- вакансии и карточки вакансий;
- пошаговые отклики кандидатов;
- вопросы кандидатов;
- сообщения через бота;
- условия работы или службы;
- фотоальбомы с отправкой изображений через MAX;
- сотрудники кадрового подразделения;
- роли и подтверждение доступа;
- принятие отклика в работу и назначение ответственного;
- импорт вакансий с портала «Работа России»;
- архивирование откликов, вопросов и сообщений;
- восстановление записей из архива;
- полное удаление записей с разграничением прав;
- переход от вакансии к откликам по этой вакансии;
- настройка организации, публичных текстов и цветов интерфейса;
- раздел «О программе»;
- проверка обновлений и обновление с GitHub;
- автозапуск через systemd.

## Требования

- Debian или Ubuntu Server;
- Python 3;
- доступ в интернет для установки зависимостей и работы с MAX API;
- токен MAX-бота для запуска бота.

## Быстрая установка

```bash
git clone https://github.com/Ilmet90/max-hr-recruitment-bot.git
cd max-hr-recruitment-bot
sudo bash install.sh
```

Установщик задаёт вопросы и разворачивает проект в `/opt/max-hr-recruitment-bot`.

## Что спросит установщик

- полное название организации;
- краткое название организации;
- вышестоящая организация;
- регион;
- название бота;
- логин главной web-учётной записи;
- пароль главной web-учётной записи;
- служебный код для команды `/admin`;
- `MAX_BOT_TOKEN`;
- `APP_HOST`;
- `APP_PORT`.

Пароль, служебный код и `MAX_BOT_TOKEN` вводятся обычным видимым текстом.

Можно оставить пустыми:

- пароль — установщик сгенерирует его и покажет один раз в конце установки;
- служебный код `/admin` — установщик сгенерирует его и покажет один раз в конце установки;
- `MAX_BOT_TOKEN` — web-админка установится, а токен можно будет указать позже в `/opt/max-hr-recruitment-bot/.env`.

## Где открыть web-админку

После установки откройте:

```text
http://<IP_сервера>:<APP_PORT>
```

Если установка выполняется в VirtualBox через NAT, настройте проброс порта. Например:

```text
host 8010 -> guest 8000
```

После этого на основной машине открывайте:

```text
http://127.0.0.1:8010
```

Если порт занят, выберите другой свободный порт.

## Первый вход

Войдите под логином главной web-учётной записи, который был указан при установке. Пароль задаётся при установке или генерируется автоматически, если поле было оставлено пустым.

Главная учётная запись нужна для первичной настройки и аварийного доступа, если сотрудники ещё не подтверждены или потеряли доступ.

## Настройка организации

После первого входа откройте «Настройки организации» и проверьте:

- полное и краткое название организации;
- вышестоящую организацию;
- регион;
- название бота;
- приветствие кандидатов;
- текст после отклика, вопроса и сообщения;
- предупреждение о персональных данных;
- контакты;
- заголовки web-админки;
- основные цвета интерфейса.

Новые значения используются в web-админке, приветствии бота, сообщениях кандидату, условиях и контактах.

## Настройка MAX-токена

Если токен был указан при установке, служба бота запускается автоматически.

Если токена ещё нет:

1. Получите токен MAX-бота.
2. Откройте `/opt/max-hr-recruitment-bot/.env`.
3. Заполните `MAX_BOT_TOKEN`.
4. Запустите службу:

```bash
sudo systemctl start max-hr-bot.service
sudo systemctl enable max-hr-bot.service
```

## Регистрация сотрудников

Сотрудник пишет боту:

```text
/admin <служебный код>
```

Заявка получает статус «Ожидает подтверждения». Главная учётная запись или начальник кадрового подразделения подтверждает пользователя в web-админке или через служебное меню `/staff`.

После подтверждения сотрудник получает логин и временный пароль для web-админки через MAX.

## Роли

- Главная учётная запись — полный доступ и аварийный web-вход.
- Начальник кадрового подразделения — подтверждает сотрудников, управляет ролями, назначает отклики, сбрасывает пароли и делегирует полномочия.
- Сотрудник кадрового подразделения — принимает отклики в работу, видит свои и общие отклики, редактирует вакансии, условия, фото и контакты.
- Ожидает подтверждения — заявка на доступ ещё не одобрена.
- Отключён — доступ отключён.

## Вакансии и импорт с «Работа России»

Вакансии можно создавать вручную в web-админке или синхронизировать с портала «Работа России».

Для импорта откройте «Вакансии → Импорт с Работа России», укажите ИНН работодателя или код работодателя на портале и нажмите «Проверить вакансии». Сначала всегда показывается предпросмотр.

Повторный импорт обновляет ранее импортированные вакансии по `external_source='trudvsem'` и `external_id`. Вакансии не удаляются автоматически, если они исчезли во внешнем источнике. Вручную созданные вакансии не перезаписываются.

Адреса, координаты, контактные лица и телефоны из внешнего API не публикуются в карточках вакансий. Исходный JSON сохраняется в служебном поле `external_raw_json`, чтобы уже импортированные вакансии можно было переочистить.

Синхронизация также доступна через `/staff → Вакансии → Синхронизировать с Работа России`.

## Отклики, архив и удаление

В разделе «Отклики» доступны фильтры:

- Активные;
- Архив;
- Все.

Активные отклики показываются по умолчанию. Архив скрывает старые записи из основного списка, но не удаляет их из базы. Запись можно вернуть из архива.

Полное удаление необратимо и доступно только главной учётной записи, начальнику кадрового подразделения и временно делегированному начальнику. Обычный сотрудник может архивировать и восстанавливать записи, но не видит кнопку полного удаления.

Из раздела «Вакансии» можно перейти к откликам по конкретной вакансии по ссылке «Отклики: N». Если по вакансии есть архивные отклики, рядом отображается ссылка «Архив: N».

В служебном меню `/staff` есть пункт «Архив откликов». Через MAX можно посмотреть архив и вернуть отклик из архива. Полное удаление через MAX-бот не предусмотрено.

## Вопросы и сообщения, архив и удаление

Разделы «Вопросы» и «Сообщения» также имеют фильтры «Активные», «Архив» и «Все». Записи можно архивировать, возвращать из архива и полностью удалять при наличии прав.

Для старых или обработанных записей рекомендуется использовать архив. Полное удаление применяйте только когда запись действительно нужно удалить окончательно.

## Фотоальбомы

Фотографии условий работы или службы группируются по альбомам. Внешний пользователь открывает их через «Условия службы → Просмотреть фотографии».

Отключённые альбомы и отключённые фотографии не показываются кандидату. Локальные ссылки на файлы не отправляются: изображения передаются через MAX API.

## О программе и обновления

В web-админке есть раздел «О программе». Обычный сотрудник видит версию. Главная учётная запись, начальник кадрового подразделения и временно делегированный начальник могут:

- проверить обновления на GitHub;
- обновить проект из GitHub;
- перезапустить web-админку;
- перезапустить MAX-бота.

Аналогичный раздел есть в `/staff`. Перед использованием кнопок обслуживания настройте ограниченный sudoers:

```bash
sudo bash /opt/max-hr-recruitment-bot/scripts/setup_maintenance_sudoers.sh
```

Обновление через web-интерфейс сначала проверяет `sudoers`. Если sudoers не настроен, web-интерфейс покажет предупреждение и не остановит службы. При корректных правах обновление делает backup, синхронизирует код и перезапускает web-админку только в самом конце.

При обновлении сохраняются:

```text
.env
data/
logs/
backups/
app/static/uploads/
```

Перед заменой кода создаётся резервная копия базы и загруженных файлов.

## Команды systemd

```bash
sudo systemctl status max-hr-admin.service
sudo systemctl status max-hr-bot.service
sudo systemctl restart max-hr-admin.service
sudo systemctl restart max-hr-bot.service
sudo journalctl -u max-hr-admin.service -f
sudo journalctl -u max-hr-bot.service -f
```

## Backup

В установленном проекте:

```bash
cd /opt/max-hr-recruitment-bot
sudo bash scripts/backup.sh
```

Архив включает SQLite-базу и загруженные файлы.

## Безопасность

- Не публикуйте `.env`.
- Не храните токены и пароли в репозитории.
- Используйте сложные пароли.
- Меняйте служебный код `/admin` при необходимости.
- Не открывайте web-админку в интернет без HTTPS, VPN или другой дополнительной защиты.
- Регулярно делайте резервные копии.
- При утечке MAX-токена перевыпустите токен.
- При увольнении сотрудника отключайте его учётную запись.
- Архив — предпочтительный способ скрыть старые отклики, вопросы и сообщения.
- Полное удаление записей необратимо.
- Перед импортом из внешних API проверяйте предпросмотр.
- Не размещайте в публичных текстах персональные данные сотрудников или служебную информацию, не предназначенную для публикации.
