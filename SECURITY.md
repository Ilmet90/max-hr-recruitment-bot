# SECURITY

## Основные правила

- Не публикуйте `.env`.
- Не храните токены, пароли, временные пароли и хеши в репозитории.
- Используйте сложные пароли для главной учётной записи и сотрудников.
- Не открывайте web-админку в интернет без HTTPS, VPN или другой дополнительной защиты.
- Используйте VPN, SSH-туннель или внутреннюю сеть.
- Регулярно делайте резервные копии.
- При утечке MAX-токена перевыпустите токен и обновите `.env`.
- Регулярно меняйте служебный код `/admin`, особенно после кадровых изменений.
- При увольнении сотрудника отключайте его учётную запись и уведомления.
- Архив — предпочтительный способ скрыть старые отклики, вопросы и сообщения из основного списка.
- Полное удаление записей необратимо; используйте его только при наличии прав и понятной причины.
- Не размещайте в публичных текстах персональные данные сотрудников, сведения ограниченного доступа или служебную информацию, не предназначенную для публикации.
- Перед импортом с внешних API проверяйте предпросмотр.
- Не публикуйте в карточках вакансий контактные телефоны и персональные данные контактных лиц, если они пришли из внешнего API.
- Web-обновление и перезапуск служб требуют ограниченного sudoers-файла. Не используйте `NOPASSWD:ALL`.
- Если sudoers не настроен, web-интерфейс должен завершить обслуживание без остановки служб.
- Разрешайте через sudoers только конкретные команды `systemctl` для служб проекта.
- Обновление должно выполняться только из доверенного GitHub-репозитория.

## Права пользователей

- Главная учётная запись имеет полный доступ.
- Начальник кадрового подразделения и временно делегированный начальник могут управлять пользователями, назначать отклики и полностью удалять записи.
- Обычный сотрудник может работать с откликами и архивом, но не должен иметь право полного удаления записей.
- Ожидающие подтверждения и отключённые пользователи не должны иметь доступ к web-админке и служебному меню.

## Что делать при инциденте

1. Остановите службу бота:

```bash
sudo systemctl stop max-hr-bot.service
```

2. Перевыпустите MAX-токен и обновите `.env`.
3. Смените пароли главной учётной записи и сотрудников.
4. Смените служебный код `/admin`.
5. Проверьте список сотрудников и отключите лишние учётные записи.
6. Проверьте последние отклики, вопросы, сообщения и архив.
7. Запустите службы и проверьте логи.
