# SECURITY

## Основные правила

- Не публикуйте `.env`.
- Не храните токены, пароли и временные пароли в репозитории.
- Используйте сложные пароли.
- Не открывайте порт web-админки в интернет без HTTPS и дополнительной защиты.
- Используйте VPN, SSH-туннель или внутреннюю сеть.
- Регулярно делайте резервные копии.
- При утечке MAX-токена перевыпустите токен.
- При увольнении сотрудника отключайте его учётную запись.
- Регулярно меняйте служебный код `/admin`.
- Не размещайте в публичных текстах персональные данные сотрудников, сведения ограниченного доступа или служебную информацию, не предназначенную для публикации.
- Web-обновление и перезапуск служб требуют ограниченного sudoers-файла. Не используйте `NOPASSWD:ALL`.
- Разрешайте через sudoers только конкретные команды `systemctl` для служб проекта.
- Обновление должно выполняться только из доверенного GitHub-репозитория.
- Импорт вакансий использует публичный API портала «Работа России»; перед импортом проверяйте предпросмотр.
- Не публикуйте в карточках вакансий контактные телефоны и персональные данные контактных лиц, если они пришли из внешнего API.
- Не храните персональные данные в публичных текстах вакансий, условий и сообщений бота.

## Что делать при инциденте

1. Остановите службу бота:

```bash
sudo systemctl stop max-hr-bot.service
```

2. Перевыпустите MAX-токен и обновите `.env`.
3. Смените пароли главной учётной записи и сотрудников.
4. Смените служебный код `/admin`.
5. Проверьте список сотрудников и отключите лишние учётные записи.
6. Запустите службы и проверьте логи.
