# Инструкции для агентов

- Работать внутри папки проекта.
- Перед Python-командами активировать виртуальное окружение: `source .venv/bin/activate`.
- Если окружение не активировано, использовать `.venv/bin/python` и `.venv/bin/pip`.
- Не читать, не выводить и не изменять `.env` без отдельной команды пользователя.
- Не хранить токены и другие секреты в коде.
- Все секреты брать только из переменных окружения.
- Проверять код командой: `source .venv/bin/activate && python -m compileall app`.
- Не запускать бесконечный Long Polling без отдельной команды пользователя.
- Не удалять `.env`, `.venv/`, `data/`, `logs/`.
